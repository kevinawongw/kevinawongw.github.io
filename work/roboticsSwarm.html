description.

Our project will use terminal input to read a command. A group of ePuck robots will then create shapes and figures given the command such as “Circle", "Square", and "Triangle” Given these commands, the ePuck robots will create the shape as a group while avoiding collisions. Due to COVID-19, we were unable to implement this with physical robots, so this is implemented with a robot simulation environment called WeBots.

implementation.

Each ePuck has it's own controller and is programmed to reach a target goal. We created a dictionary where the initial pose of the ePuck is the "key", and a tuple of the target pose and it's batch number is it's "value". Robots and targets are sorted based on distance from origin do determine which ePuck should go where. Each ePuck begins their movement in 10 second intervals to avoid collision avoidance. There is also a server that communicates with all of the robots along with the user input script, so target goals can be posted. The server also allows each robot to post its coordinates and makes a cookie to keep track of the different robots. If targets have been posted and we have enough robots, the server maps each robot to a target. Robots can then make get requests for their targets and if the time for the robot to move has come and the mapping is done, they will receive a target pose.

improvement.

Look forward in this project, we would definitely try to find methods of improving ePuck accuracy when it goes to the target goals. Also, we originally planned to implement natural language processing into this project where we could have a user verbally say a shape (ie."circle") as the input. Due to time constraints, this was a feature we were unable to add, but it is something worth adding in the future. Lastly, more complex shapes and patterns could be a great addition to this project.
